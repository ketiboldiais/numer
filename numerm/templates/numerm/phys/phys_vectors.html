Problem Solving in Physics

The following problem-solving strategy is often useful for solving physics problems:

Identify explicit data. The first step is to identify all the relevant data. What are the problem's conditions? Given those conditions, what concepts apply?

Identify implicit data. Here, we want to identify any data that is implied from the explicit data. For example, if we know that ${x + 2 = 4,}$ we have the implicit data that ${x = 2.}$ Identifying these implicit data often proves enormously useful for later calculations. This is also the step where we identify things we don't know. What are the target variables? I.e., what are the things we want to find out about?

Set up the problem. This is where we want to layout all the equations, algorithms, systems, data structures, and concepts necessary to solving the problem. It may seem as if everything is relevant — with practice, this step can become much more like pattern-recognition.

Implement and execute the solution. Having laid out all of our tools, we now use them to implement a solution. Once implemented, we execute the solution and examine the results.

Test. Assuming the results returned correctly, we now test. Here, we'll want to replace the provided values with similar ones, then begin exploring edge cases.

Evaluate. Having performed all of the previous steps, we must then evaluate the answer. Think of this as writing a journal entry for the entire process, a sort of self-reflection. What is the significance for each step in the process? What are the solution's shortcomings? Were there edge cases that failed?

Modeling

One of the most valuable skills in physics is modeling — the process of reducing complex systems into smaller, simplified systems. For example, analyzing the motion of a bullet fired from a handgun is terrifyingly complex. How much gunpowder is used? What is the chemical composition of said gun powder? What is the handgun's model? Air resistance? Bullet's weight? Air temperature? Humidity? Distance? Height? All these questions are relevant, and unfortunately, paint a freakishly complex reality. If we tried solving the problem by answering all these questions at once, we likely wouldn't get anywhere.

Instead, a better approach is to model. Instead of thinking of the bullet with all its contours and metallic nature, we just envision a particle. Instead of worrying about the gun power chemistry, we just assume some initial acceleration. Instead of worrying about air resitance, we assume no air resistance; a vaccuum. We make whatever assumptions are necessary to make the system as easy as possible to analyze. This is modeling.
