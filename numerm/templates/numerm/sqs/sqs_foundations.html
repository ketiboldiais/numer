Number Sequences

A number sequence is an ordered list of numbers. In computer science, we might think of that sequence as an array. The sequence's order is defined by a rule. The rule might be described symbolically (to be discussed momentarily) or with words. For example, the rule “The sequence starts at 1 and each term is 2 more than the previous term” yields the sequence: ${\lang 1, 3, 5, 7, 9, 11, \ldots \rang.}$

Some sequences carry on forever. We call such sequences infinite sequences. For example, the sequence of natural numbers, ${\langle 0, 1, 2, 3, \ldots \rangle}$ is a sequence that goes on forever. In contrast, other sequences are finite. For example, the sequence of all natural numbers less than 5, ${\langle 0, 1, 2, 3, 4 \rangle,}$ is a finite sequence.

As we might notice, English is a poor language for describing number sequences. Even the slightest complexity can lead to descriptions that are difficult to state and parse. Considering this difficulty, we often want to either: (a) list all of the terms (for finite sequences); (b) listing the first few terms and assuming the pattern continues indefinitely with ellipses (a dangerous tactic); (c) using a formula to represent the ${n^{\text{\scriptsize{th}}}}$ term, or general term; (d) using a recurrence formula to describe the ${n^{\text{\scriptsize{th}}}}$ term; of (e) representing the sequence with a graph.
