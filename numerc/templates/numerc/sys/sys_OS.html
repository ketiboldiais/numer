The Operating System

A computer system consists of four parts: (1) the hardware, (2) the operating system, (3) application programs, and (4) the user. So far, we've focused heavily on application programs. Application programs are those we're all familiar with: compilers, browsers, text editors, games, etc. In this section, we focus on the operating system. An operating system is a program that manages a computer system's hardware. That management mandates two roles: (1) serving as a regulator and gatekeeper for connections between programs and hardware, and (2) providing an interface between the user and the hardware. To perform these dual roles, operating systems consist of numerous pieces. We examine each of these pieces in turn.

In the hardware section, we presented the view of thinking of hardware as a large city where bits are transported from place to place, pursuant to the user's commands. The operating system is akin to this city's government. As a government, the operating system has numerous roles. It enforces laws for what programs can and cannot do. It regulates how I/O devices connect to hardware. It also serves as a resource allocator: CPU time, memory space, storage space, power usage, I/O devices, etc.

The operating system itself is often identified by a program called the kernel. This is a program that runs at all times on a computer system. We can think of the kernel as the heart of the operating system. The user never actually interacts with the kernel. Instead, the user writes programs, and the programs request “permission” from the kernel to execute the user's instructions. We do not refer to the kernel as the operating system because the operating system does not just consist of a kernel. It also consists of system programs — programs that provide environments, or platforms, for other programs to run. Modern operating systems also often include middleware — programs that provide additional tools for software development on the operating system.
