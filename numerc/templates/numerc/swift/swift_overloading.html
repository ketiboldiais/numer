Function Overloading. In Swift, functions can be overloaded — the phenomenon of when multiple functions share the same name. Because these functions all have the same name, the compiler must have some way to tell them apart. To provie the compiler that information, we provide the functions different parameters in the parameter list. 


		func foo(a: Int) -> String {
			if a == 0 {
				return "Zero"
			} else { return "Not zero" }
		}
		
		func foo(b: Int) -> String {
			if b == 1 {
				return "One"
			} else { return "Not one" }
		}
		
		print(foo(a: 0))
		print(foo(b: 1))
	

		Zero
		One

Among the above approaches, if we overload a function based on type, we must explicitly state those types when calling the function. This is because type inference will not apply to the function call.

A good example of a built-in overloaded function is the stride() function. This is similar to the range() method in Python. 


		for i in stride(from: 10, to: 0, by: -2) {
			print(i)
		}
	

		10
		8
		6
		4
		2

The from: parameter represents the first element of the interval, to: represents the last element of the interval, and by: represents the distance between each element in the interval.

Overloads should be used with care. With multiple functions sharing the same name, we decrease maintainability. In situations where overloading is desired, we should always consider whether a default parameter would be a better idea.
