stacks. We can think of a View as block of Lego. With smaller blocks, we can build bigger and more complex structures. This idea extends to Views. We can collect and arrange Views to build bigger and more complex Views, culminating in a user interface — the struct named ContentView. There are three primary ways to arrange Views: through a VStack, an HStack, and a ZStack.

The VStack arranges its collection of Views vertically. The HStack arranges it collection of Views horizontally. The ZStack arranges its views toward the user — i.e., along the Z-axis, views on top of the views, moving up and off the screen. Remember, all of these — VStack, HStack, and ZStack — are themselves Views.
