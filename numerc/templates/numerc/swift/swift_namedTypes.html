Named Types

In Swift, there are two kinds of types — named types and compound types. A named type is a type that has a name when it's defined. Swift provides four named types — struct (structures), class, enum (enumerations), and protocol.

The built-in types we've encountered so far are actually named types — Int, String, Bool, Array, Dictionary, Set, etc.

In contrast to named types, a compound type has a type signature when it's defined. A tuple in Swift is a compound type. For example, the tuple (1, 2, 3) has the type signature (Int, Int, Int). The tuple ("Logged in", true) has the type signature (String, Bool). The tuple (true, (1, 1.18)) has the type signature (Bool, (Int, Double)). In Swift, functions are a compound type. We explore functions in the next section.

In addition to the distinction between named types and compound types, types are also distinguished between value types and reference types. Value type determines how a datum is stored, and reference type determines how a datum is copied. So far, much of the types we've worked with are value types. 


		var x = 0
		var y = x
		y = 1
		var isXequaltoY = (x == y)
		print(isXequaltoY)
	

		false

In the code above, we created an instance of the value type Int, namely x, with the value 0. Then, we assigned that instance, x, to the instance y. Both x and y are completely independent (this should look familiar to those coming from Java). When we wrote y = 1, both x and y no longer match. Why? Because they are now instances of a different value type. The types struct and tuple are value types.

Reference types are something entirely different. With a reference type, the two instances are both referencing the same data. This means that if make a change to one instance, we make a change to the other instance. In Swift, a class and a function are examples of reference types.
