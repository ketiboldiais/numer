Variables

In Swift, a variable is a name that refers to an object. This may familiar to those coming from Java. The variable itself does not store the object, but a reference to the object. The object to which variable refers to is called the variable's value. There are two ways to initialize or declare a variable in Swift:

var is short for “variable.” The keyword let is more interesting. The word has traditionally been used in mathematics and physics to denote constants or base assumptions, making its way into the BASIC programming language in 1964, then to functional programming languages in the 1970s, all the way down to Swift and many others. Other languages have value-related keywords such as where, def, const, and val. Like JavaScript, we have two ways to declare a variable: (1) with the atom let, or (2) with the atom var. However, unlike JavaScript, these two atoms have very specific meanings that must be adhered to strictly. First, let is used to initialize a constant, and var is used to initialize a variable. Which of let and var should be used depends on the problem and the programmer's style. For those coming from a functional programming background, let should be the preferred choice, so as to avoid mutability (a constant cannot be mutated; once declared, it is assigned that value permanently). For those coming from an object-oriented programming background, var may or may not be the desired choice (a variable can be mutated). 


		var x = 10
		let z = 5

		// this is ok - you can mutate x - returns 15
		x = x + 5

		// this is bad - you cannot mutate a constant - returns an error
		z = z + 1

Both constants and variables must have the same type as the value assigned to the variable. This type can never change. For example, if we write var x = 5, then the variable x will always store a value of type Int; x can never store any other value of a different type, but it can store a different value of type Int. 


		let implicitInt = 1
		let implicitFloat = 1.0

We can be more explicit with constants and variables by specifying its type: 


		let explicitInt: Int = 1
		let explicitFloat: Float = 1.0

The colon is read "of type." Thus, writing var x: Int tells Swift, "Create a variable named x of type Int." And if we wrote let y: Double, we tell Swift, "Create a constant named x of type Double. It is not always necessary to explicitly specify a variable or constant's type. This is because Swift provides type-inference (i.e., if we simply wrote let x = 100, Swift can infer that x is a constant of type Int).

Declaring variables, we may see a warning stating “Variable ... was never mutated; consider ...” This is Swift's way of reminding us we should err on using constants whenever possible. Mutability is a double-edged sword. Whenever we declare a variable with var, we add to the list of values we have to keep track of. Too long a list, and we increase the chances of bugs appearing.

Variable & Constant Names. Names in Swift can consist of ASCII and Unicode characters (this includes numbers, as long as they are not the first character in the name). What they cannot contain: whitespace, math symbols (e.g., +, -, *, /, %, etc.), as well as certain other characters that are seldom ever used as variable names in the real world (XCode will notify us when we violate these rules). In addition to these exclusions, Swift subscribes to camelCase convention. Once a variable is declared, it cannot be "redeclared." In other words: 


		// This will not work
		var x = 5
		var x = 6

		// But this will
		var x = 5
		x = 6
