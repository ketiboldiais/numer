Algorithms for Strings

In this section, we present several useful algorithms for strings. As we know, a character set is the set of character codes a language can process. Because computers only understand zeroes and ones, each character in the set is assigned an integer value, which can be represented in zeroes and ones. These codes are determined by an encoding standard. In languages like C, the the encoding standard is ASCII.

For the ASCII characters, the encodings are the integers from ${0}$ to ${127.}$ This is a total of ${128}$ unique encodings, thereby permitting the representation of ${128}$ unique characters. Because the highest possible integer representation is ${127}$ — in binary, ${111~1111_{[2]}}$ — seven bits are sufficient for representing ${128}$ characters. However, because computer memory (RAM), is addressed by byte (eight bits), a minimum of 1 byte is allocated for the ASCII encoding — in C and C++, a char value. Because every ASCII encoding requires exactly 1 byte, we say that ASCII uses a fixed-width encoding system.

This is a good point to introduce an important distinction: There's a difference between the character set (ASCII) and the character set's encoding system. The encoding system is the way the characters in the set are represented in memory. In ASCII, characters are encoded as seven-bit character codes, as we stated earlier. There are, however, other encoding systems:

  • In UTF-32, “UTF” stands for “Unicode Transformation Format.” all characters are encoded as a sequence of 32-bits (4 bytes). As such, UTF-32 is a fixed-width encoding system. The predominant character set today is Unicode. The name “Unicode” is a morphological blending of unique, unified, universal, and encoding. As of this writing, Unicode is currently at version 14.0, comprising ${144~697}$ characters, with ${1~112~064}$ code points in reserve.
